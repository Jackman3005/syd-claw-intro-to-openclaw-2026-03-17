The Workspace

What makes your agent yours

~/.openclaw/workspace/
├── SOUL.md ← personality, tone, boundaries
├── AGENTS.md ← operating instructions
├── USER.md ← who you are, your preferences
├── MEMORY.md ← long-term memory (agent-maintained)
├── TOOLS.md ← your setup specifics
├── HEARTBEAT.md ← periodic check instructions
└── skills/ ← modular capabilities

Same model, same code, completely different agent.
Two people running OpenClaw will have totally different experiences.

Context Is Everything

The model is the same. The context makes it yours.

💬
Zero context
"What should I have for dinner?"
Generic answer. Model knows nothing about you.
🔄
Conversation context
20 messages in, you mentioned you're vegetarian
Better — but resets next session.
📌
Light personalization
ChatGPT memory, Claude projects
A few bullet points persist. Shallow, not yours to control.
🦞
Full context injection
SOUL.md, USER.md, AGENTS.md, tools, history...
Deep, persistent, and you control all of it.

The model isn't smarter — it just knows more about you before it starts thinking.
Same engine, different onboarding docs → completely different employee.

OpenClaw vs Coding Agents

💻 Claude Code / Codex

  • Lives in your terminal
  • You open it, use it, close it
  • Scoped to your codebase
  • No persistent memory
  • No messaging integration
  • Amazing at coding

🦞 OpenClaw

  • Always running on your machine
  • Messages you on WhatsApp/Discord
  • Knows about your life, not just code
  • Persistent memory across sessions
  • Cron jobs, heartbeats, automation
  • Can spawn coding agents inside it

They're complementary, not competing. You can run Claude Code as a sub-agent inside OpenClaw.

Getting Started

~5 minutes to your first conversation

1
Install
npm install -g openclaw
Node.js 20+. Mac, Linux, WSL2, Raspberry Pi.
2
Setup
openclaw setup
Creates config + workspace. Add your API key (Anthropic, OpenAI, etc).
3
Connect a channel
openclaw channels login
Scan QR for WhatsApp, paste token for Telegram/Discord.
4
Start the Gateway
openclaw gateway
Your agent is now live. Send it a message! 🦞

Security Considerations

It has real access — treat it accordingly

🔒
Local-first
Your data stays on your machine. Only external calls are to your LLM provider.
👤
allowFrom
Restrict who can talk to your agent. Don't run open-to-the-world.
🛡️
Tool policies
Control what commands the agent can run. Sandbox mode available.
🔑
Gateway auth
Token-protected WebSocket. Especially important for remote access.
📁
Workspace = memory
Treat workspace files like personal data — because they are.
🐢
Start conservative
Lock it down first, open up as you build trust with the agent.
